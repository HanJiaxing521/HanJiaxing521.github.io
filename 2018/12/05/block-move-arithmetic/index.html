block move arithmetic

2018-12-05

The author is a sophomore in the School of Information and Software Engineering at the University of Electronic of Science and Technology of China. I am interested in algorithm research and focuses on machine learning. When I participated in the 2019 Microsoft Imagine Cup, I encountered a problem that needed to solve the problem of determining which attributes of a group of samples were representative. I accidentally thought of an algorithm, but I don��t know if the predecessors had created it, or the algorithm is less feasible. In short, let’s share it today, welcome everyone to correct me.
The specific problem is this:
Now that there are a group of patients with unknown infectious diseases, doctors want to find out their common characteristic diseases for research, so he is ready to screen out the representative properties of these diseases.
Problem abstraction:
There is a set of samples that are known to have similarities, but I don’t know where they are. The number of attributes is known, the boundaries of the various attributes are known, and the attributes of the set of samples have similarities.
Algorithm process:
Sort the data in each attribute from small to large (assuming there are 10 data in total, plus 12 data in the upper and lower bounds of the data):

Num1, num2, num3…num10

  1. Divide the interval into 12 blocks, one for each block��
  2. Of course, the distribution of data will not be so average, there must be concentration and dispersion, then move the data along with the data block back to the real position:
  3. If the distance between the data is close, the database squeezes each other, so ��blank�� is left in the interval. The larger the blank, the more concentrated the data is. Then we calculate the top k attributes with the total length of the gap. It is a representative attribute.
  4. Method of calculating the blank length: When the average distribution data is in the middle of the data block, the relative distance between the data when the data block is moved is the blank area left.

code show as below:

#To prevent the original data set from being modified, use a copy for subsequent operations.
def mainFeature(dataset, labels, r, k):
sortDataset = dataset
sortDataset.sort(axis=1) #Sort the data in each dimension from small to large
numDimen = sortDataset.shape[0] #Calculating dimensions
featureCount = {}

#Calculate the space occupancy of each dimension:
for i in range(numDimen):
    numDataIndimen = sortDataset[i].shape[0]
    dimenSize = sortDataset[i][-1] - sortDataset[i][0]
    segmentSize = dimenSize / numDataIndimen  #Calculation section                                               length
    blankSize = 0
    #Calculate the blank length
    for j in range(numDataIndimen):
        if j == (numDataIndimen - 1):
            break
        if (sortDataset[i][j+1] - sortDataset[i][j]) > segmentSize:
            blankSize += sortDataset[i][j+1] - sortDataset[i][j] -segmentSize
    #Calculate the white space rate and add items larger than r to a dictionary
    blankSpaceRate = blankSize / dimenSize
    if blankSpaceRate > r:
        featureCount[labels[i][0]] = blankSpaceRate

#Count the top k dimensions (ie, features) with the largest blank rate and return
sortedFeatureCount = sorted(featureCount.items(), key=operator.itemgetter(1))
sortedFeatureCount = sortedFeatureCount[::-1]

sortedFeature = np.array([])
for i in range(k):
    sortedFeature = np.append(sortedFeature, sortedFeatureCount[i][0])

return sortedFeature
